# Как сделать bitaps VPN реальным приложением

Сейчас на маке стоят только Command Line Tools — нативный апп без полного Xcode не собрать.
Ниже три уровня: от «нативный апп в руках» до «реально подключает VPN».

---

## Уровень 1 — настоящий нативный апп (на моках, как прототип, но в Xcode/на iPhone)

1. **Поставить Xcode** — App Store, бесплатно (~10 ГБ). После установки открыть его один раз, принять лицензию.
2. Переключить инструменты:
   ```bash
   sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
   ```
3. Открыть проект:
   ```bash
   open /Users/kashtanka/bitaps-vpn-app/BitapsVPN.xcodeproj
   ```
4. Подписать: таргет **BitapsVPN → Signing & Capabilities → Team** = твой Apple ID
   (бесплатный годится для симулятора и своего iPhone на 7 дней).
5. Сверху выбрать схему **BitapsVPN**, устройство (iPhone-симулятор или **My Mac**) → **⌘R**.

→ Запустится настоящее приложение: все экраны, навигация, подключение **на моках**.
Можно поставить на свой iPhone и щупать вживую. Это уже «реальное приложение», просто VPN пока имитируется.

> Если Xcode покажет ошибки сборки — пришли мне текст, поправлю (я проверял синтаксис,
> но полную типизацию без Xcode не прогнать).

## Уровень 2 — VPN реально подключает

Нужно:
1. **Сервер с VLESS + Reality** (РУ-нода на Яндексе). Конфиг сервера — в `server/singbox-server.json` (готов, подставь свои ключи).
2. **libbox** — gomobile-сборка sing-box в `Libbox.xcframework`. Скрипт: `scripts/build-libbox.sh`.
   Положить xcframework в таргет **PacketTunnel**.
3. Включить **App Group** `group.app.bitaps.vpn` на всех таргетах (в Signing & Capabilities).
4. В `Sources/Core/Services/VPNService.swift` поставить `TunnelFactory.useSingBox = true`,
   а реальные ключи прокинуть в `SingBoxTunnel.makeConfig(for:)`.

---

## Уровень 3 — реальные аккаунты/подписки

1. **Supabase** (проект `bjkozsukvifkxriojxrz`, тот же что у бота): дать anon key.
2. В `Secrets.supabaseAnonKey` вписать ключ, поставить `APIFactory.useSupabase = true`.
3. Эндпоинты (servers / subscription / devices / issue-config) — схема в `server/supabase.sql`.

---

## Что нужно от тебя (Варвары)
- **Apple ID** (бесплатный — для теста; **$99/год Apple Developer** — для NetworkExtension и публикации в App Store).
- **Сервер** (Яндекс) + параметры VLESS+Reality: host, port, UUID, public key (pbk), short id (sid), SNI.
- **Supabase** anon key.
- Решить **bundle id** (сейчас `app.bitaps.vpn`).

Дай это — и я доведу до реального подключения. Без сервера/ключей дальше уровня 1 не уехать.
